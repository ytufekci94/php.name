PHP Global Variable

PHP superglobals are a set of predefined variables that are available in all scopes throughout a PHP script. These variables are called "superglobals" because they are accessible from anywhere in the script, regardless of the context or scope.


There are eight superglobals in PHP:

  • *$_GET: An array of variables passed to the script via the HTTP GET method.
  • *$_POST: An array of variables passed to the script via the HTTP POST method.
  • *$_COOKIE: An array of variables stored in the user's browser as cookies.
  • *$_FILES: An array of variables related to uploaded files.
  • *$_REQUEST: An array that combines the variables of the $_GET, $_POST, and $_COOKIE arrays.
  • *$_SESSION: An array of variables stored in a session.
  • *$_ENV: An array of variables passed to the script via the environment.
  • *$_SERVER: An array of variables related to the server and execution environment.

$_GET

Here's an example of how you might use the $_GET superglobal:

Example

<?php

if (isset($_GET['name'])) {
  $name = $_GET['name'];
  echo "Hello, $name!";
}

?>

In this example, the $_GET array is checked for the presence of a "name" element. If the element exists, its value is stored in the $name variable and used to greet the user.

It's important to note that superglobals are not automatically sanitized or validated, so it's up to the developer to ensure that the data is safe and appropriate for use.

Example

<?php

if (isset($_POST['submit'])) {
  $name = $_POST['name'];
  $email = $_POST['email'];
  $message = $_POST['message'];

  // Validate and sanitize the data here

  // Save the data to a database or send an email, etc.
}

?>

In this example, the $_POST array is checked for the presence of a "submit" element. This element is typically used as a submit button in a form, and its presence indicates that the form has been submitted. If the element exists, the script retrieves the values of the "name", "email", and "message" elements and stores them in variables.

It's important to note that superglobals are not automatically sanitized or validated, so it's up to the developer to ensure that the data is safe and appropriate for use. In this example, the data should be validated and sanitized before it is saved or used in any way.

$_COOKIE

here's an example of how you might use the $_COOKIE superglobal

Example

<?php
if (isset($_COOKIE['name'])) {
  $name = $_COOKIE['name'];
  echo "Welcome back, $name!";
} else {
  echo "Welcome, guest!";
}

?>

In this example, the $_COOKIE array is checked for the presence of a "name" element. If the element exists, its value is stored in the $name variable and used to greet the user. If the element does not exist, the user is greeted as a guest.

To create a cookie, you can use the setcookie() function, like this:

<?php

setcookie('name', 'John', time()+3600);


?>

This creates a cookie named "name" with a value of "John" that will expire in one hour.

It's important to note that superglobals are not automatically sanitized or validated, so it's up to the developer to ensure that the data is safe and appropriate for use.

$_FILES

here's an example of how you might use the $_FILES superglobal

Example

<?php
if (isset($_FILES['file'])) {
  $file = $_FILES['file'];
  $fileName = $file['name'];
  $fileType = $file['type'];
  $fileError = $file['error'];
  $fileSize = $file['size'];
  $fileTemp = $file['tmp_name'];

  // Validate and sanitize the file here

  // Move the file to a permanent location
  move_uploaded_file($fileTemp, "uploads/$fileName");
}

?>

In this example, the $_FILES array is checked for the presence of a "file" element. If the element exists, the script retrieves various properties of the file and stores them in variables. These properties include the file name, type, error code, size, and temporary location.

It's important to note that the $_FILES array only contains data for files that have been uploaded via a form with the enctype="multipart/form-data" attribute.

It's also important to note that superglobals are not automatically sanitized or validated, so it's up to the developer to ensure that the file is safe and appropriate for use. In this example, the file should be validated and sanitized before it is moved to a permanent location.

$_SESSION

here's an example of how you might use the $_SESSION superglobal

Example

<?php
// Start a new session or resume an existing one
session_start();

// Set a session variable
$_SESSION['name'] = 'John';

// Check if a session variable is set
if (isset($_SESSION['name'])) {
  $name = $_SESSION['name'];
  echo "Hello, $name!";
}

// Destroy the session
session_destroy();

?>

In this example, the session_start() function is used to start a new session or resume an existing one. The $_SESSION array is then used to set a session variable named "name" with a value of "John".

The isset() function is used to check if the "name" session variable is set. If it is, the value of the variable is retrieved and used to greet the user.

The session_destroy() function is used to destroy the current session. This will remove all session variables and end the session.

It's important to note that superglobals are not automatically sanitized or validated, so it's up to the developer to ensure that the data is safe and appropriate for use.
